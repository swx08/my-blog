为什么要使用限流？限流有什么好处？ ​

使用限流（Rate Limiting）在分布式系统、微服务架构以及API服务中是非常重要的，主要有以下几个原因和好处：

  1. 防止服务过载： 当系统面临大量的并发请求时，如果不加以限制，可能会导致系统资源耗尽（如CPU、内存、数据库连接等），进而引发服务响应延迟、超时甚至宕机。通过限流，可以确保系统的负载在可承受的范围内，避免服务过载。
  2. 保护后端资源： 后端资源（如数据库、缓存、消息队列等）通常都有处理能力的上限。如果前端服务不加限制地发送请求到后端，可能会超出后端资源的处理能力，导致后端资源崩溃。通过限流，可以控制发送到后端资源的请求速率，保护后端资源不被压垮。
  3. 防止恶意攻击： 在API服务中，恶意用户可能会通过发送大量的请求来尝试耗尽服务资源，造成拒绝服务攻击（DoS）或分布式拒绝服务攻击（DDoS）。通过限流，可以限制单个用户或IP地址的请求速率，防止恶意攻击对系统造成影响。
  4. 提高系统稳定性： 通过限制请求速率，系统可以更加稳定地运行。当系统面临突发流量时，限流可以确保系统能够平滑地处理这些流量，而不是因为突然的过载而崩溃。此外，限流还可以帮助系统在高负载下保持一定的响应能力，为用户提供更好的服务体验。
  5. 优化资源分配： 在微服务架构中，不同的服务可能具有不同的处理能力和资源需求。通过限流，可以根据服务的实际处理能力来分配请求流量，确保每个服务都能得到足够的资源来处理请求，避免资源分配不均导致的性能问题。
  6. 实现服务降级： 在分布式系统中，当某个服务出现故障或性能下降时，通过限流可以将对该服务的请求限制在一个较低的水平，避免因为该服务的故障而影响到整个系统的稳定性。同时，还可以结合熔断机制，将故障服务进行隔离，确保其他服务的正常运行。

因此，使用限流可以提高系统的稳定性、可靠性和安全性，是分布式系统和API服务中不可或缺的一种技术手段。

有哪些限流框架呢？ ​

  1. Guava RateLimiter

    • Google的Guava库提供了一个简单的RateLimiter类，用于限制某个操作的速率。
    • 它基于令牌桶算法，允许你指定每秒或每分钟等时间间隔内允许的最大请求数。

  2. Sentinel:

    • 阿里巴巴开源的分布式系统的流量防卫兵。
    • 提供了丰富的流量控制、熔断降级、系统自适应保护等多个功能。
    • 支持实时动态调整限流策略。

  3. Resilience4j:

    • 另一个流行的容错库，提供了限流、熔断、重试等功能。
    • 它与Java 8的函数式接口和lambda表达式完美融合。

  4. Hystrix:

    • Netflix开源的熔断器库，虽然已经被宣布为"不再活跃"的项目，但在过去的很长一段时间内，它都是微服务架构中广泛使用的熔断和限流库。

  5. Bucket4j:

    • 一个基于Java的令牌桶算法实现，用于限制对资源（如API接口）的访问速率。
    • 它提供了可配置性强、可扩展性好的限流方案。

  6. SimpleRateLimiter:

    • 如果你的需求比较简单，只是需要一个轻量级的限流工具，可以考虑使用SimpleRateLimiter这样的库。

redis配置 ​

java
/**
 * redis序列化
 */
@EnableCaching
@Configuration
public class Redis extends CachingConfigurerSupport {

    @Bean
    public RedisTemplate<Object, Object> redisTemplate(RedisConnectionFactory connectionFactory) {
        RedisTemplate<Object, Object> redisTemplate = new RedisTemplate<>();

        //默认的Key序列化器为：JdkSerializationRedisSerializer
        redisTemplate.setKeySerializer(new StringRedisSerializer()); // key序列化
        redisTemplate.setValueSerializer(new GenericJackson2JsonRedisSerializer()); // value序列化

        redisTemplate.setConnectionFactory(connectionFactory);
        return redisTemplate;
    }
}

Lua脚本

lua
-- 获取KEY，针对哪个接口限流。lua脚本的数组默认索引是1而不是0
local key = KEYS[1]
-- 获取注解上标注的的限流值
local limit = tonumber(ARGV[1])

local currentLimit = tonumber(redis.call('get', key) or "0")

-- 超过限流次数直接返回0(触发限流机制)，否则走else分支
if currentLimit + 1 > limit
then return 0

--首次直接进入
else
    -- 否则，将当前次数+1，并设置过期时间
    redis.call('INCRBY', key, 1)
    -- 设置过期时间 秒
    redis.call('EXPIRE', key, ARGV[2])
    return currentLimit + 1
end

自定义注解 ​

java
/**
 * 自定义限流注解
 * @author weixueshi
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface RedisLimitAnnotation {

    /**
     * 资源key(唯一)
     * 作用：标记不同的接口，不同的流量控制
     */
    String key() default "";

    /**
     * 提示信息
     */
    String message() default "当前访问人数过多，请稍后再试！";

    /**
     * 过期时间，单位秒（默认30）
     */
    long expireTime() default 30;

    /**
     * 最多访问限制次数
     */
    long maxRestrictions() default 3;
}

切面类 ​

java
/**
 * 限流切面
 */
@Slf4j
@Aspect
@Component
public class RedisLimitAspect {

    Object result = null;

    @Resource
    private StringRedisTemplate stringRedisTemplate;

    private DefaultRedisScript<Long> redisLuaScript;

    /**
     * 加载lua脚本
     */
    @PostConstruct
    public void init(){
        redisLuaScript = new DefaultRedisScript<>();
        redisLuaScript.setResultType(Long.class);
        redisLuaScript.setScriptSource(new ResourceScriptSource(new ClassPathResource("rateLimiter.lua")));
    }

    /**
     * 环绕通知
     * @param joinPoint 切点
     * @return Object
     */
    @Around("@annotation(org.common.annotation.RedisLimitAnnotation)")
    public Object redisLimitAround(ProceedingJoinPoint joinPoint) throws Throwable {
        //获取方法上的注解
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        Method method = signature.getMethod();
        RedisLimitAnnotation limitAnnotation = method.getAnnotation(RedisLimitAnnotation.class);

        if(limitAnnotation != null){
            //获取注解的参数
            //获取限流的唯一key
            String key = limitAnnotation.key();
            String className = method.getDeclaringClass().getName();
            String methodName = method.getName();

            //构造redis的key
            String limitKey = key + "\t" + className + "\t" + methodName;
            log.info("限流key：{}",limitKey);

            //key必须设置
            if(key == null){
                log.error("限流key不能为空");
                throw new SystemException(ResponseCodeEnum.KEY_NOT_EXIST);
            }

            //获取最大访问次数
            long maxRestrictions = limitAnnotation.maxRestrictions();
            //获取过期时间
            long expireTime = limitAnnotation.expireTime();

            List<String> keys = new ArrayList<>();
            keys.add(key);

            Long count = stringRedisTemplate.execute(
                    redisLuaScript,
                    keys,
                    String.valueOf(maxRestrictions),
                    String.valueOf(expireTime));

            log.info("当前访问次数：{}；key：{}",count,key);
            if(count != null && count == 0){
                log.error("触发限流机制！{}",key);
                return ResultData.fail(ResponseCodeEnum.FAIL.getCode(),limitAnnotation.message());
            }
        }

        //执行目标方法
        result = joinPoint.proceed();
        return result;
    }
}
